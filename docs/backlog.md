# Product Backlog
Project: Kangina – Smart Restaurant Ordering System

---

## User Story 1: User Registrationde
As a new user, I want to register an account so that I can access the system features.

**Priority:** High  
**Story Points:** 3  

**Acceptance Criteria:**
- User can input name, email, and password  
- System validates required fields  
- Account is successfully created  

---

## User Story 2: User Login
As a user, I want to log in so that I can access my account and place orders.

**Priority:** High  
**Story Points:** 3  

**Acceptance Criteria:** 
- User can enter email and password
- System validates credentials
- User is redirected afer login

## User Story 3: Browse Available Dishes
As a user, I want to browse a variety of dishes online so that I can easily choose what I want without going to a carinderia store physically.

**Priority:** High  
**Story Points:** 5  

**Acceptance Criteria:**
- Users can view list of available dishes with names, prices, and images
- Users can filter, search, and categorizes dishes for quick browsing
- Users can click on a dish to see detailed information

## User Story 6: Ratings and Reviews
As a user, I want to be able to rate and give my reviews to the foods I have bought.

**Priority:** High  
**Story Points:** 5  

**Acceptance Criteria:**

## User Story 7: Manage Menu and Food Availability
As an admin, I want to manage the menu and food availability so that customers can see accurate and available dishes for ordering.

**Priority:** High  
**Story Points:** 5  

**Acceptance Criteria:**
- Admin can add new menu items with name, price, description, and image
- Admin can edit or remove existing menu items
- Admin can update dish availability (e.g., available / unavailable)
- Changes made by the admin are immediately reflected in the user interface
